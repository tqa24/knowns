# Auto-Sync

Knowns can resync generated skills and instruction files after CLI upgrades.

---

## What Auto-Sync Covers

Auto-sync is about generated AI-facing assets, not project data:

- built-in skills
- instruction files such as `CLAUDE.md`, `OPENCODE.md`, `GEMINI.md`, `AGENTS.md`
- compatibility shims generated by `knowns sync`

---

## Manual Sync

Current CLI commands:

```bash
# Sync skills and instruction files
knowns sync

# Sync skills only
knowns sync --skills

# Sync instruction files only
knowns sync --instructions
```

Sync always overwrites generated files to keep them in sync with the current CLI version. The `--force` flag is deprecated and no longer needed.

For instruction-file generation only, `knowns agents --sync` is also available as a compatibility command.

---

## Config Interaction

Project config can limit active platforms with `settings.platforms` and can enable upgrade-driven syncing with `settings.autoSyncOnUpdate`.

Example:

```json
{
  "settings": {
    "platforms": ["claude-code", "copilot", "agents"],
    "autoSyncOnUpdate": true
  }
}
```

---

## Practical Notes

- Sync always overwrites generated files (skills, instruction files) to match the current CLI.
- If a platform is filtered out by config, `knowns sync` skips it.
- If docs and generated files disagree after an upgrade, rerun `knowns sync` and trust the current CLI output.
